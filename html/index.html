Comandos básicos GIT por Alisson

Comandos Quando e como usar O que faz
git config
$git config --global user.name "Seu nome aqui"
$git config --global user.email "usuario@email.com" 		git config deve ser usado para fazer
a configuração do usuário no git bash, deve ser passado exatamente o mesmo usuário e senha usado no github 		Configura o git no computador do
desenvolvedor(a), passando nome e email do github para que possa ser iniciado um novo repositório na máquina do desenvolvedor(a).
git init
$git init 		Inicia um repositório onde estiver sendo usado o git bash Cria um pasta oculta com o nome de .git, demonstrando que ali agora possui um novo repositório git.
git add
$git add .
$git add caminho/nomeDoArquivo.arquivo 		Deve ser usado quando algum arquivo do projeto for alterado ou criado e houver a necessidade de enviar essas alterações ao repositório git. O git add . adiciona todas as alterações efetuadas de uma vez só para fazer o commit.
O outro git add adiciona apenas o arquivo que foi passado o nome e/ou o caminho completo do arquivo.
git commit
git commit -m
git commit -a -m
 Após o comando git add ter sido feito, é hora de enviar as alterações feitas para o repositório git. Ao realizar o comando git commit, ele ainda não será enviado ao repositório git, ficará apenas no repositório local até que seja feito um git push.Todo commit realizado exige que haja pelo menos uma mensagem, essa mensagem descreve brevemente o que foi feito. O git commit -m, passa apenas a mensagem descrevendo a alteração, há um outro comando alternativo, o git commit -a -m, esse comando adiciona todos arquivos que foram modificados/criados e também já passa a mensagem que descreve o que foi feito.
git push
$git push
$git push origin NomeDaBranch 		Após ser feito o git add e o git commit, agora as modificações devem ser enviadas ao repositório git, deixando essas mudanças de serem apenas "local". $git push envia todos os commits que estão local para a branch que o usuário estiver trabalhando. $git push origin NomeDaBranch, envia os commits para a branch NomeDaBranch no repositório git.
git pull
$git pull
$git pull origin NomeDaBranch 		Deve ser usado sempre que houver uma nova versão de uma branch específica, ou caso o desenvolvedor tenha feito um commit de uma branch e deseja voltar a ela. Faz a "cópia" de tudo que está na branch no repositório git ou no repositório local e caso haja alguma diferença entre a branch que o desenvolvedor se encontra, com a que foi feita o pull as modificações serão adicionadas à branch do desenvolvedor.
git clone
$git clone "URL do repositório" 		Deve ser usado sempre que o desenvolvedor desejar clonar um projeto que esteja no repositório git, criando um repositório local. Cria um repositório local do repositório git do qual foi passado a url.
git checkout
git checkout Branch01
git checkout -b Branch02 		Sempre que for necessário trocar de uma branch para outra o comando git checkout deve ser executado. O comando git checkout Branch01, muda o desenvolvedor de sua branch atual para a Branch01, o comando git checkout -b Branch02, cria uma nova branco com o nome Branch02, e já muda automaticamente para essa branch.
git stash
git stash show
git stash list
git stash apply
